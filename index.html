Built Weenix, a mini-Unix operating system, in the CSCI 1690 Operating Systems course at Brown. The project is split into 5 parts:

Procs: Processes, threads and mutexes
Drivers: Device drivers for terminals, disks and memory devices
Virtual File System (VFS): Common interface between operating system kernel and various file systems
System V File System (S5FS): File System implementation based on the original Unix file system
Virtual Memory (VM): Implement virtual memory maps, page fault handler, anonymous and shadow objects, and systems calls such as fork(). With these, the Weenix kernel is able to manage user address spaces, run user-level code and service system calls.
